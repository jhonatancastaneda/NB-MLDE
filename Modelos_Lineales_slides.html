Qué es R ?

  • R es un dialecto de S

Qué es S ?

  • S es un lenguaje de programación desarrollado por Jhon Chambers en Bell Labs en 1976.

  • Inicialmente se generó como un entorno estadístico interno (implementado en Fortran).
  • En 1988 es reescrito en lenguaje C (versión 3)
  • En 1998 se lanza la versión 4, que actualmente se maneja.
  • Bell Labs –> Insightful Corp (licencia, lenguaje, 2 M.)
  • En 2004 Insightful añade GUIs y varías características, S-PLUS

  • En 2008 Insightful Corp –> TIBCO (lenguaje, 25 M)

Historia de R

  • Fue desarrollado en 1991 por Ross Ihaka y Robert Gentleman en la Universidad de Auckland.
  • En 1993 se anuncia al público en general.
  • En 1995, Martin Mächler convence a Ross y Robert de distribuir a R mediante la licencia GNU (General Public License).
  • EN 1996, se crean las listas de correo públicas R-help y R-devel.
  • En 1997 se forma el R Core Group o R Core Team, encargado de controlar el código fuente de R.
  • En el 2000 se publica la primera versión de R 1.0.0.
  • En marzo 2015 se publica la versión 3.1.3.

Diseño del sistema R

El sistema de R puede considerarse como la interacción de dos componentes principales:

1. Sistema Base de R.

  • El paquete base de R contiene las funciones fundamentales que permiten correr el programa en sí.

Adicionalmente, al momento de instalar el programa se cargan por default paquetes estándar que permiten ampliar la capacidad de los gráficos, funciones estadísticas clásicas entre otros.

Instalando paquetes

Una de las principales ventajas de R es su capacidad para incrementar su funcionalidad mediante la incorporación de nuevos paquetes o librerías.

  • Para instalar un paquete se utiliza la linea de código:
install.packages("pckgname")

Ciertos paquetes de R requieren la instalación de otros paquetes adicionales debido que entre estos comparten algunas funciones, por lo cual es necesario adicionar el siguiente parámetro:

install.packages("pckgname", dependencies=TRUE)

En el caso que se desee instalar un lista de \(n\) paquetes planteamos la siguiente solución:

paquetes <- c("pckg_1","pckg_2", ... , "pckg_n")
install.packages(paquetes, dependencies=TRUE)
# funciones apply
# sapply(X=paquetes, FUN=install.packages, dependencies=TRUE)

Ejemplos

  • Para instalar el paquete de nombre "devtools", utilizamos la línea de código:
install.packages("devtools", dependencies = TRUE)
  • Para instalar los paquetes "foreign", "forecast", "gWidgets", utilizamos:
paquetes <- c("foreign", "forecast", "gWidgets")
install.packages(paquetes, dependencies=TRUE)
# funciones apply
# sapply(X=paquetes, FUN=install.packages, dependencies=TRUE)

Instalando paquetes

install.packages("pckgname", repos="http://www.omegahat.org/R", type="source")

Git & Github

  • GitHub es un repositorio de archivos y proyectos el cual emplea un sistema de control de versiones conocido como Git y permite almacenar el código en repositorios de forma pública.

  • La instalación y sincronización de Git & Github con RStudio se realiza de la siguiente manera:

    • Descargar e instalar la plataforma específica de Git con las opciones por default.
    • Configurar Git con los comandos globales a través de la versión bash, esto permitirá que Git tenga conexión con los repositorios de Github para el envió/recepción de archivos, teclear lo siguiente:
  git config --global user.name "nombre de la cuenta de Github"

  git config --global user.email "Github-email@something.com"
  • Por último, abrir RStudio y configurar la ruta del ejecutable de Git: Ir a Tools > Options > Git/SVN.

Estructuras de Datos

  • Las estructuras de datos en R se organizan por su dimensionalidad y tipo de elementos (Homogeneos y heterogeneos):
Dimensión Homogeneos Heterogeneos
1d Atomic vector List
2d Matrix Data Frame
nd Array

  • Observación. R no tiene estructuras 0d (magnitudes escalares), los números y strings se consideran como vectores de longitud 1

Vectores

  • La estructura de datos básica de R son los vectores, estos se dividen en:
    • Vectores atómicos y Listas
  • Presentan tres propiedades comunes:
    • Tipo (Qué es?). typeof()
    • Longitud (Número de elementos) length()
    • Atributos (Metadata). attributes()
  • Para probar que una estructura de datos es un vector (atómico o lista) se utiliza la función:
is.vector(x)
  • is.vector(x) retorna TRUE si x es un vector atómico o una lista.

Vectores atómicos

  • Los elementos de un vector atómico son del mismo tipo (tipos comunes):
    • logical
    • integer
    • double (comunmente llamados numeric)
    • character
  • Un vector es creado mediante la función c(), e ingresamos a la i-ésima componente con x[i]
dbl_var <- c(1, 2, -1) 
int_var <- c(3L, 7L, 1L) # con el sufijo L se declaran números enteros
log_var <- c(FALSE, TRUE, F, T) # TRUE y FALSE (T, F) se usan para crear vectores lógicos
chr_var <- c("Source", "Stat", "Lab")
# tercera componente de dbl_var
dbl_var[3]
c(1, c(2, c(3,4))) ; c(1, 2, 3, 4) # los vectores pueden ser anidados
## [1] 1 2 3 4
## [1] 1 2 3 4

Tipos y Tests

  • Dado un vector se puede determinar su tipo con typeof() o verificar un tipo específico con una "is" función:
    • is.character()
    • is.double()
    • is.integer()
    • is.logical()
dbl_var <- c(1, 2, -1)
typeof(dbl_var)
## [1] "double"
is.double(dbl_var)
## [1] TRUE

Tipos y Tests 

dbl_var <- c(1, 2, -1)
is.atomic(dbl_var)
## [1] TRUE
int_var <- c(3L, 7L, 1L)
typeof(int_var)
## [1] "integer"
is.integer(int_var)
## [1] TRUE
is.atomic(int_var)
## [1] TRUE

Tipos y Tests

  • is.numeric() retorna TRUE para vectores double e integer
dbl_var <- c(1, 2, -1)
is.numeric(dbl_var)
## [1] TRUE
int_var <- c(3L, 7L, 1L)
is.numeric(int_var)
## [1] TRUE

Coerción

  • Los elementos de un vector son del mismo tipo, es por eso que si combinamos tipos diferentes, serán coercionados al tipo más flexible dado por la jerarquía:

    • character <= double <= integer <= logical
x <- c("ssl", 1, TRUE)
str(x)
##  chr [1:3] "ssl" "1" "TRUE"

  • Se usa las funciones "as" para coercionar un vector a un determinado tipo: as.character(), as.double(), as.integer(), as.logical()

  • Las funciones +, abs, log, etc coercionan a double o integer

  • Las funciones &, |, any, all, etc coercionan a logical

x <- c(1L, 3L) ; y <- c(2L, 5L) ; str(x + y)
##  int [1:2] 3 8

Coerción 

x <- c(FALSE, TRUE, FALSE, TRUE) 
as.numeric(x) # TRUE coerciona a 1 y FALSE a 0
## [1] 0 1 0 1
sum(x) # número total de TRUEs 
## [1] 2
mean(x) # proporción de TRUEs
## [1] 0.5

Listas

  • Una lista puede contener elementos de cualquier tipo.
  • Para construir una lista se utiliza la función list() en lugar de c().
x <- list(1:3, c("Source", "Stat", "Lab"), c(TRUE, FALSE), c(1.3, 4.5))
# El operador `:` se usa para generar secuencias enteras
str(x)
## List of 4
##  $ : int [1:3] 1 2 3
##  $ : chr [1:3] "Source" "Stat" "Lab"
##  $ : logi [1:2] TRUE FALSE
##  $ : num [1:2] 1.3 4.5
is.atomic(x)
## [1] FALSE
# coercionamos un objeto a una lista
as.list(c("ssl", 4, 0.5))

Listas 

x <- list(1:3, c("Source", "Stat", "Lab"), c(TRUE, FALSE), c(1.3, 4.5))
typeof(x)
## [1] "list"
# probamos si x es una lista
is.list(x)
## [1] TRUE
# pasamos a vector considerando las reglas de coerción
unlist(x)
##  [1] "1"      "2"      "3"      "Source" "Stat"   "Lab"    "TRUE"  
##  [8] "FALSE"  "1.3"    "4.5"

Listas

  • Una lista es un vector recursivo, es decir una lista puede contener otra lista (diferencia principal con los vectores atómicos)
x <- list(list(list("Source", "Stat")))
str(x)
## List of 1
##  $ :List of 1
##   ..$ :List of 2
##   .. ..$ : chr "Source"
##   .. ..$ : chr "Stat"
# son equivalentes. un vector no puede contener otro vector
c(1, c(2, c(3,4)))  ;  c(1, 2, 3, 4)
## [1] 1 2 3 4
## [1] 1 2 3 4

Elementos de una lista

  • Para acceder al elemento i de la lista se utiliza x[i], en cambio para acceder al objeto que contiene el elemento i se utiliza x[[i]] o x$nomb_obj.
x <- list(1:3, c("Source", "Stat", "Lab"), nomb_obj=c(TRUE, FALSE), c(1.3, 4.5))
x[4]
## [[1]]
## [1] 1.3 4.5
x[[4]]
## [1] 1.3 4.5
x$nomb_obj
## [1]  TRUE FALSE

Atributos

  • Los objetos en R poseen varios atributos tales como:
    • nombres
    • clase
    • dimensiones, etc
  • La función attributes() muestra los atributos de un objeto.
x <- c(1:5)
attributes(x)
## NULL
x <- list(nomb_obj1=1:3, nomb_obj2=c("Source", "Stat", "Lab"), nomb_obj3=c(TRUE, FALSE))
attributes(x)
## $names
## [1] "nomb_obj1" "nomb_obj2" "nomb_obj3"

Matrices

  • Una matriz es un vector con el atributo dimensión
  • La dimensión es un vector de longitud 2: c(nrow, ncol)

A

Missing Values

  • Los valores pérdidos en R se denotan por NA, NaN.
  • NA Not Available. Dato pérdido.
  • NaN Not a Number. Resultado de una indeterminación.
    • NaN es NA, el reciproco no se cumple.
x <- c(Inf*0, 3.2, NA, 5, NA)
x
## [1] NaN 3.2  NA 5.0  NA
x <- c(-1, 0/0, NA, 5, -6)
x
## [1]  -1 NaN  NA   5  -6

Missing Values

  • is.na(). prueba si un objeto es NA.
  • is.nan(). prueba si un objeto es NaN.
x <- c(Inf*0, 3.2, NA, 5, NA)
is.na(x)
## [1]  TRUE FALSE  TRUE FALSE  TRUE
x <- c(-1, 0/0, NA, 5, -6)
is.nan(x)
## [1] FALSE  TRUE FALSE FALSE FALSE
  • Inf representa \(\infty\)

Sentencia if

  • Se utiliza para testear una proposición y realizar una determinada operacion

  • Se utiliza para testear una proposición y realizar una determinada operación

Subsetting

  • la función subset() se utiliza para seleccionar elementos de un objeto.
x <- 1:5
y <- subset(x,subset = x>3)
y <- x[x>3]
y
## [1] 4 5
x <- c(1, 8, NA, 3, NA)
y <- subset(x,subset = !is.na(x))
y <- x[!is.na(x)]
y
## [1] 1 8 3
